2  * \brief Java Microbenchmarking Harness
3  *
4  * <h1>Setup</h1>
5  * <ul>
6  * <li>To run this harness one will have to run it using maven</li>
7  * <li>Please verify that you have maven installed by running <i>mvn -version</i> from command line</li>
8  * <li>Add the new sample to com.freemindcafe.jmh.sample<n> package</li>
9  * <li>Let's assume that the project location on your machine is /path/to/samples-jmh</li>
10  * <li>Run '<i>mvn clean install</i>' command from /path/to/samples-jmh</li>
11  * <li>This will generate the benchmark code under /path/to/samples-jmh/target/generated-sources</li>
12  * <li>Jmh uses APT (annotation processing tool) to generate source by reading the annotations</li>
13  * <li>Run the specific benchmark that you want to verify by running the command below</li>
14  * <li><i>java -jar target/benchmarks.jar <your benchmark class name regular expression> -wi 5 -f 1 -i 5</i></li>
15  * </ul>
16  * <p>
17  * To know all the harness options please refer to http://java-performance.info/jmh/
18  * </p>
19  *
20  * <h1>Terminology</h1>
21  * <ul>
22  * <li>Benchmark annotations (@Benchmark) are processed independently unless we group two benchmarks by creating a group.</li>
23  * <li>warmup iterations (-wi 5) are run that number of times per benchmark annotation.</li>
24  * <li>warmup iteration time (-w 5) is the time for which per warmup iteration will be run.</li>
25  * <li>iterations (-i 5) are number of times a benchmark will be run.</li>
26  * <li>iteration time (-r 5) is the duration in seconds for which a single iteration will be run.</li>
27  * <li> Number of threads (-t 5) are the threads in which the benchmark code will be executed concurrently.</li>
28  * <li>Fork option (-f 2) forks a new jvm. And every warmup iteration and measurement iteration is run independently in those vms.</li>
29  * </ul>
30  *
31  *
32  */
